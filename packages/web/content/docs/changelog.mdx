## Release notes

Full release notes for every published version of Agent Orchestrator are available on the GitHub releases page:

**[https://github.com/ComposioHQ/agent-orchestrator/releases](https://github.com/ComposioHQ/agent-orchestrator/releases)**

Each release entry includes:
- A summary of new features and improvements
- Bug fixes
- Breaking changes (if any), with migration guidance
- The set of packages that changed and their new versions

---

## Versioning policy

Agent Orchestrator uses [Semantic Versioning](https://semver.org/) (semver):

| Version bump | When |
|---|---|
| **Patch** (`x.y.Z`) | Bug fixes, non-breaking internal changes |
| **Minor** (`x.Y.0`) | New features, new plugin APIs, backwards-compatible changes |
| **Major** (`X.0.0`) | Breaking changes to public APIs or config schema |

Versions are managed with [Changesets](https://github.com/changesets/changesets). Every pull request that changes behavior must include a changeset file (`.changeset/*.md`) describing the change and its impact level.

## Per-package changelogs

This is a monorepo. Each package maintains its own `CHANGELOG.md` at the package root. The most relevant ones:

| Package | Changelog location |
|---|---|
| `@aoagents/ao-core` | `packages/core/CHANGELOG.md` |
| `@aoagents/ao-cli` | `packages/cli/CHANGELOG.md` |
| `@aoagents/ao-web` | `packages/web/CHANGELOG.md` |
| `@aoagents/ao` | `packages/ao/CHANGELOG.md` |
| Plugin packages | `packages/plugins/<name>/CHANGELOG.md` |

These files are auto-generated by Changesets on each release and committed to the repository.

## Checking your installed version

```bash
ao --version
```

To check versions of individual packages when running from source:

```bash
node -e "console.log(require('./packages/ao/package.json').version)"
```

---

## Updating to the latest release

If you installed `ao` globally:

```bash
pnpm update -g @aoagents/ao
```

If you are running from the repository source:

```bash
git pull origin main
pnpm install
pnpm build
```

After any update, run config validation to catch schema changes early:

```bash
ao config validate
```

See [Migration Notes](/docs/migration) for guidance on handling breaking changes between versions.
