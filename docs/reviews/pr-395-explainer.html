What this PR does

  • Adds stable event_id to OpenClaw escalation payloads.
  • Includes Event ID: <id> in escalation text for traceability.
  • Implements AO-side idempotency dedupe keyed by sessionKey + event_id with TTL via idempotencyTtlMs (default 300000 ms).
  • Prevents timeout/retry replay duplicates by reserving keys before outbound send.

Architecture and design choices

  • Phase-0 mitigation is client-side only, matching issue scope.
  • Dedupe store is in-memory map with lazy TTL eviction for low overhead.
  • Scoping by session key prevents cross-session suppression of identical event IDs.
  • post() path keeps deterministic hash-based event IDs but bypasses escalation dedupe policy.
